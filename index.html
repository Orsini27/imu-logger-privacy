Privacy Policy – IMU Logger App

Last updated: 19 January 2026

1. Introduction

IMU Logger App is a mobile application designed for the collection and analysis of human motion data using inertial sensors (IMU) available on Android devices. The application is intended for research, clinical, and academic purposes, particularly in the context of gait analysis and movement asymmetry studies.

This Privacy Policy explains what data is collected, how it is used, how it is stored, and what rights users have, in accordance with the General Data Protection Regulation (GDPR) and Google Play policies.

2. Data Collected

2.1 Participant Profile Data

When the application is used for the first time, the participant may be asked to provide the following information:

  • Age
  • Height (in centimeters)
  • Gender (Male, Female, None of the options, Prefer not to say)
  • Dominant leg (Left or Right)

This information is used exclusively to improve the interpretation of biomechanical data and can be edited later through the application interface.

2.2 Sensor and Motion Data

During data recording sessions, the application may collect:

  • Accelerometer data
  • Gyroscope data
  • Magnetometer data
  • Linear acceleration
  • Gravity vector
  • Orientation (quaternions)
  • Step detection events
  • Relative timestamps

These data are strictly related to motion analysis and do not contain direct personal identifiers.

2.3 Technical and Calibration Data

The application may also store technical metadata such as:

  • Device identifier (Android device ID)
  • Device placement (e.g., pocket location)
  • Initial foot used during calibration

The device identifier is a technical identifier provided by the Android system and does not reveal the user's identity, phone number, or contact information.

3. Data Storage

3.1 Local Storage

Collected data are initially stored locally on the device in CSV files. Each file contains metadata in the first line followed by time-series sensor data.

3.2 Remote Storage

When an internet connection is available, CSV files may be uploaded automatically to a remote storage service (Supabase Storage) for backup and further analysis.

4. Purpose of Data Processing

The collected data are used exclusively for:

  • Scientific and academic research
  • Human movement and gait analysis
  • Biomechanical pattern analysis
  • Detection and study of movement asymmetries
  • Development, testing, and validation of motion analysis algorithms

The application does not provide medical diagnoses and is not intended to replace professional clinical assessment.

The data are not used for advertising, marketing, or commercial profiling.

5. Data Sharing

The application does not sell or share data with third parties for commercial purposes. Access to the data is limited to authorized researchers or developers involved in the project. Any publication or analysis uses anonymized or aggregated data whenever possible.

6. Legal Basis

Data processing is based on:

  • User consent (Article 6(1)(a) GDPR)
  • Scientific research purposes (Article 9(2)(j) GDPR)

7. Data Retention

Data are retained only for as long as necessary to fulfill the purposes described in this policy. Users may request data deletion where technically and legally possible. Uninstalling the application removes locally stored data from the device.

8. User Rights

In accordance with GDPR, users have the right to:

  • Access their data
  • Request correction of inaccurate data
  • Request deletion of data
  • Withdraw consent at any time
  • Restrict or object to data processing

9. Data Security

Reasonable technical and organizational measures are implemented to protect user data, including local application storage, secure transmission, and restricted access. However, no method of electronic storage or transmission is completely secure.

10. Children

The application may be used by minors only under the supervision of a legal guardian, particularly in clinical or research settings.

11. Changes to This Policy

This Privacy Policy may be updated periodically. Any changes will be reflected on this page with an updated revision date.
